「Webコンテンツ作成」の成果

本テーマについて

この授業では、HTML、CSS、JavaScriptを用いて、動きのあるWebページを作成する技術を学びました。基本的なHTML構造から始まり、 CSSでの装飾、JavaScriptによる動的な処理、DOM操作による要素の追加・変更・削除までを段階的に実践しました。 最終的には、フォームの操作やイベント処理、データの表示など、実用的なWebコンテンツを自らの手で構築できるようになることを目指しました。

プロジェクト課題について

このプロジェクト課題では、NHKの番組表を検索できるウェブページを作りました。ユーザーがチャンネルとジャンルを選び、 「検索」ボタンを押すと、その条件に合った番組が表示されます。データはインターネット上のJSONファイルから取り、JavaScriptを使ってページに表示します。 検索キーが「NHK総合1-映画」「NHK総合1-劇場・公演」「NHK総合1-福祉」「Eテレ1-スポーツ」「Eテレ1-映画」「Eテレ1-劇場・公演」「Eテレ1-趣味・教育」であるときに、 データが無いから「番組が見つかりませんでした」と表示されます。また、前の検索結果を消してから新しい結果を出すようにして、画面が見やすくなるようにしました。 この課題を通して、HTML、JavaScript、データの取り扱い方を学びました。

プロジェクト課題で作成したページ: NHK番組表の検索

数当てゲーム

このゲームでは、HTML と JavaScript を使って作成し、 1〜10の中からランダムに選ばれた整数を当てるシンプルな数当てゲームです。 第一回作成するときにテキストボックスからの入力やボタンによるイベントの処理，結果をページに表示する方法など，習っていなかったので、 予想値を4に固定して、予想回数は最大3回で、コンソールに正解やヒント、ゲーム終了のメッセージが表示されます。 第二回になると、完成品をできました。 完成品では、プレイヤーはテキストボックスに数字を入力し、「回答」ボタンをクリックして予想します。予想できる回数は最大3回までで、正解かどうか、 または答えに近いかどうかのヒントが表示されます。ゲームが終了すると、正解だったか、間違えたかの結果メッセージが表示され、再度予想することはできません。 HTMLでは入力欄やボタン、結果表示用の段落タグ（<p id="result">）を用意し、JavaScriptではイベント処理とDOM操作を使ってゲームを完成させました。

各授業回の学習事項

第1回 WWW と HTML

今回の授業では、HTMLの基本構造と要素について学びます。 Webページの作成のためのHTML文書の書き方、要素の意味や使い方、属性の指定方法、 特殊文字の扱いなどを理解し、実際にコードを書いてWebページを作成できるようになることを目指します。

演習の成果:演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

今回の授業では、HTMLで記述されたWebページに対して、CSSを用いて見た目を装飾する方法を学びます。 文字の色やサイズ、配置、背景色、枠線の設定など、 また、外部スタイルシート、内部スタイルシートの違いや使い分けについても理解できました。

演習の成果:演習2-3: CSS の編集

第3回 JavaScript の基本

今回の授業では、JavaScript の基本構文やエラー処理、オブジェクトの使い方、デバッグ方法を学びます。 Java との違いを比較しながら、変数の扱いや関数の定義、配列やオブジェクトの操作方法について実践的に理解を深めます。 また、Firefox の開発ツールを用いたエラー確認やブレークポイントによるデバッグ技法を体験しました。

演習の成果:演習3-6: オブジェクトの参照

第4回 イベントの処理と DOM(1)

今回の授業では、HTMLのフォーム要素とJavaScriptによるイベント処理、DOM操作の基本を学びます。 具体的には、tashizan.htmlとtashizan.jsを用い、2つの数値を入力して「計算」ボタンを押すと、 足し算の結果がページ上に表示されるような仕組みを実装します。これにより、ユーザー操作に応じた動的なWebページの作成方法を習得します。

演習の成果:演習4-10: 足し算ページ

第5回 DOM(2)

今回の授業では、DOMを使ってWebページの構造をJavaScriptで動的に 操作する方法を学びました。具体的には、要素の新規作成・配置、属性の変更、要素の削除、複雑な要素の追加などを実践しました。 ボタン操作によって、画像の表示、リンクの追加、リストの更新などができるようになりました。

演習の成果:演習5-7 DOMプログラミング(2)

まとめ

この「Webコンテンツ作成」の授業では、HTML・CSS・JavaScript の基本から応用までを段階的に学び、 最終的には実際に動くWebページを自分で作る力を身につけることができました。授業は全6回にわたり、 少しずつ複雑な動きや機能を持つWebページを作ることができるようになりました。数当てゲームやproject課題やindexページの制作を通して、 HTMLで構造を作り、CSSで見やすく整え、JavaScriptで動きをつけるという一連の流れを、何度も試行錯誤しながら経験できたことは大きな学びとなりました。

工夫点としては、CSSを使ってページの見た目を整えるときに、理想とするデザインを実現するために多くの時間を使って試行錯誤したことです。 背景色は白に近い淡いブルーを選び、文字のサイズや行間、要素の余白などを調整して、画面全体が読みやすくなるように工夫しました。また、検索フォームや番組表示部分の配置も整え、 どの情報がどこにあるかすぐ分かるように心がけました。細かい見た目の調整に時間はかかりましたが、そのおかげでスッキリとした印象のページに仕上がったと思います。

苦心点は、検索結果の表示処理とエラー時の対応です。はじめは検索ボタンを押すたびに結果が重なって表示されてしまったため、JavaScriptで旧結果を自動的に削除するように処理を追加しました。 また、一部のチャンネルやジャンルの組み合わせでは番組データが存在しないケースがあり、そのままでは表示が壊れたり、エラーが出たりしていました。 そこで、データが存在しない場合には「該当する番組が見つかりませんでした」と表示させるように条件分岐を組み込みました。 こうした細かい調整に時間はかかりましたが、実用的で安定したWebページを作るために大切なステップだったと感じています。

強調点は、全体の見やすさを意識したデザインとリンクの扱いです。文字と文字の間隔を広めにとり、情報が詰まりすぎないようにしました。 配色は派手すぎず、柔らかい色合いにして、読みやすさを重視しました。さらに、ページ内にあるすべてのリンクは新しいタブで開くように設定し、ユーザーが元のページを見失わないように配慮しました。 また、リンクの内容が一目でわかるように、リンクに title 属性を設定することで、マウスを乗せたときに簡単な説明が表示されるようにしました。 こうした点は、見た目のデザインだけでなく、使いやすさの向上にもつながると考えています。

感想として、この授業を通して、Webページの構造やJavaScriptの使い方、データ処理の流れについて多くのことを学べました。はじめはうまく動かないことも多く、 エラーが出て原因がわからず困る場面もありましたが、少しずつ調べたり試したりする中で理解が深まりました。 特に、APIからデータを取得して画面に表示する一連の流れを実装できたことは、大きな達成感を感じました。 今後は、今回学んだ知識を活かして、より多機能で見やすく、使いやすいWebページを作っていきたいと思います。
